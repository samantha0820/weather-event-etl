5-Day Weather & Event ETL Pipeline and Streamlit Application

Author

Samantha Wang

1 Abstract

This project presents the development of an end-to-end ETL (Extract, Transform, Load) pipeline and a Streamlit web application that deliver daily weather forecasts and event recommendations for New York City. Using Prefect for orchestration and automation, the pipeline fetches and processes 5-day weather forecasts from the OpenWeatherMap API and upcoming events from the Ticketmaster API. The processed data are validated, merged, saved to GitHub daily, and visualized through an interactive Streamlit dashboard. The application enables users to explore daily weather summaries and recommended local events based on weather conditions. The final solution integrates automation, real-time data refresh, and cloud-based visualization, demonstrating the principles of reproducible and automated data science workflows.

2 1. Introduction

The purpose of this project is to automate the extraction and presentation of weather forecasts and event information, and make it easily accessible through a user-friendly web application. Weather conditions can heavily influence participation in local events; therefore, integrating weather and event data enables better decision-making for event-goers.

3 2. Methods

3.1 2.1 Data Sources

  • Weather Data: OpenWeatherMap 5-day forecast API
  • Event Data: Ticketmaster Discovery API

3.2 2.2 ETL Pipeline

  • Extract:
    • Fetch daily weather and event data using API keys.
  • Transform:
    • Clean, validate, and standardize the datasets using Pandera.
    • Assign recommendations (e.g., “Recommended (Outdoor)”) based on weather attributes.
  • Load:
    • Save processed CSV outputs to GitHub.
    • Automate with Prefect deployment scheduled to run daily at 1:00 AM.

3.3 2.3 Streamlit Application

  • Build an interactive dashboard to display:
    • Weather summaries by day
    • Event lists filtered by recommendation and date
  • Use Streamlit’s caching mechanism to refresh data daily.

4 3. Results

4.1 3.1 ETL Outputs

The ETL pipeline successfully produces two updated CSV files every day: - weather_forecast.csv - events_forecast.csv

4.2 3.2 Streamlit Dashboard

The Streamlit dashboard provides: - Temperature, humidity, wind, rain chances for each day - Recommended events, with clickable event links - Filtering options by event type and recommendation status

5 4. Discussion

The integration of Prefect and GitHub ensures full automation without manual intervention. The Streamlit application remains synchronized with the latest data via caching mechanisms. Future improvements could include extending the app to multiple cities, adding weather warnings, or integrating user geolocation to personalize the recommendations.

Challenges encountered included API rate limiting and timezone alignment (UTC vs. local event times), which were addressed through local normalization and selective data retrieval.
